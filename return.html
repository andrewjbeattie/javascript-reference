Return

The return statement stops the execution of a function and returns a value from that function.

The return statement ends function execution and specifies a value to be returned to the function caller.

When you have a return in your function, this will be the last line of code executed in that function.

Which means if you have a console.log on the line after the return, it will not be executed. Thus you will not see its result.

So the take away is that console.log does NOT replace return. You can have both, the console.log is optional and useful for debugging, but you really need to have the return

Examples of Mistakes

Forgetting return = result is undefined. Even though you will see 8 printed in the Console, the function double is missing a return this it will return nothing (more specifically undefined).


// THIS IS WRONG
function double(number) {
console.log(number * 2)
}

let result = double(4) // result is undefined

This also does not work, even though you will see 8 printed in the Console.
That's because return console.log will return undefined because the result of console.log is undefined.


// THIS IS WRONG
function double(number) {
return console.log(number * 2)
}

let result = double(4) // result is undefined

 The function WORKS but we don't see the console.log becayse it is placed after the return


// The function WORKS but we don't see the console.log
function double(number) {
return number * 2 // "result" of the function, will exit the function
console.log(number * 2) // we don't see this
}
let result = double(4) // result is 8 - but nothing logged in the Console

Needs to be re-written. You need to swap the return and console.log like this:


function double(number) {
console.log(number * 2)
return number * 2
}
let result = double(4) // result is 8 - we also see 8 in the Console


Sample Usage

With every function you write, there are many ways on how to call that function.


function sum(a, b) {
return a + b
}

// Sample usage (do not modify)
console.log(sum(3, 5)) // will log 8 on the console
console.log(sum(4, 1)) // will log 5 on the console

Beware of hardcoding!!!


We have sum(3, 5) as a sample usage but that does NOT mean that sum should always return 8. No! Beware of hardcoding your answer.

You should still use variables and not rely on the actual values of these variables.

This way the function is reusable over and over again.

Hardcoding is bad because it is inefficient. We want the function to be reusable over and over again with different arguments. Thus parameters should be symbolic rather than hardcoded values.

Here we ask for the character length of the different arguments from the "name" parameter.
It should return 3 for first name and 6 for second:


function myFunction(name) {
return(name.length);
}

// Sample usage (do not modify)
console.log(myFunction("Sam"))
console.log(myFunction("Oakley"))
